# Support

## Getting Help

- **Documentation**: Start with the [README](README.md), the [onboarding guide](onboarding/README.md), and the [hooks guide](hooks/README.md)
- **Bug Reports**: [Open an issue](https://github.com/jeremylongshore/governed-second-brain-plugin/issues/new?template=bug_report.md)
- **Feature Requests**: [Open an issue](https://github.com/jeremylongshore/governed-second-brain-plugin/issues/new?template=feature_request.md)
- **Contributing**: See [CONTRIBUTING.md](CONTRIBUTING.md) — it also maps which change belongs in which repo
- **Security Issues**: See [SECURITY.md](SECURITY.md) — report privately, do NOT open a public issue

## Response Times

| Channel           | Response Time   |
| ----------------- | --------------- |
| Security reports  | 24 hours        |
| Bug reports       | 3 business days |
| Feature requests  | 1 week          |
| General questions | 1 week          |

## Before Opening an Issue

1. Search [existing issues](https://github.com/jeremylongshore/governed-second-brain-plugin/issues) first
2. Check the [README](README.md) and [onboarding guide](onboarding/README.md)
3. Note which **mode** you're in (`local` by default, or `team` when `TEAMKB_API_URL` is set) — behavior differs
4. Include version info (`gsb.lock.json` engine pins), your OS, and reproduction steps
